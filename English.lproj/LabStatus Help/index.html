LabStatus

LabStatus is a small but useful tool for monitoring a set of hosts. For example you have a lab setup with 20 hosts. Each host can either boot Windows or boot linux for administrative tasks. Maybe you want to know how many hosts are up and also which kind of operating system is running. So, this is what LabStatus can do for you. In LabStatus you can define your set of hosts with the hostname, ip-address and mac-address. For each of these hosts all indicators you define will be executed. It is also possible to configure actions, for example wake on lan.

Adding hosts:

Before LabStatus can do anything you must define host entries. For each host the indicators and actions will be generated in the main window.

To add an host:

  1. Open the preferences panel.
  2. Hit the add button (+) to add an new host
  3. Enter a hostname, IP-address, MAC-address, username and password

Adding indicators:

You can define indicators for your hosts. An indicator consist of a name and a command to be executed on a host. The return value of the command will be used to change the color of the indicator. A result of 0 means success (green indicator). A result greater than 0 indicates a failure (red indicator). The string '1>/dev/null 2>&1' will automatically added to the command string during runtime.

To add an indicator:

  1. Open the preferences panel.
  2. Hit the add button (+) to add an new indicator
  3. Enter a name and a command to execute
    4. Example: Create a new indicator with the name 'PING' and enter the following command: 
    /sbin/ping -t 2 -c 1 %ipaddress%
    You can use %ipaddress%, %macaddress%, %hostname%, %username% or %password% to define your commands.

Adding actions:

You can define actions that will be executed if you hit the corresponding action button next to a host. The string '1>/dev/null 2>&1' will automatically added to the command string during runtime.

To add an action:

  1. Open the preferences panel.
  2. Hit the add button (+) to add an new action
  3. Enter a name and a command to execute
    4. Example: Create a new action with the name 'WOL' and enter the following command: 
    /usr/local/bin/wol %macaddress%
    You can use %ipaddress%, %macaddress%, %hostname%, %username% or %password% to define your commands.
